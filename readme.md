# Who is my Counselor?
학생의 진로 상담 내역 및 진로 이해 여부를 기반으로 진로 상담 선생님을 배정하는 앱입니다.

## 개발 취지
  일반적으로 고등학교에서의 진로상담은 담임교사나 진로 담당 교사가 주도합니다. 그러나 **학생이 진로 상담을 신청할 때에는 질문이 모호하거나 단순히 "진로에 대해 궁금하다"는 식으로 요청**하는 경우가 많습니다. 따라서 학생이 자신의 진로에 대해 어느 정도 이해를 하고 있는지 파악하기 위해서는 실제 상담을 진행해야만 합니다.  
  이런 류의 상담을 진행하다보면 **교사는 뒤늦게 "이건 전문가나 해당 교과 담당 선생님이 하는 것이 맞다"는 깨달음을 가지게 됩니다만, 이 통찰을 얻기까지 꽤나 많은 시간이 소요됩니다.** 내담자인 학생과 대화를 해야 이와 같은 인사이트가 도출되기 때문이죠. 뒤늦게 교사는 "이 내용은 다른 선생님께 문의하는 것이 맞다"고 학생에게 안내하겠지만 '처음에 그 선생님께 보낼걸'이라는 생각이 머리를 떠나지 않을겁니다. 학생 또한 적합하지 않은 선생님께 상담을 신청했다는 점에서 불편함을 느끼겠죠.  
  이와 같은 문제를 해결하기 위해 본 프로젝트에서는 **학생이 제출한 진로에 대한 발화를 기반으로 상담을 해야하는 교사가 누구인지 (담임이나 진로교사 vs 교과 담당 교사나 전문가) 자동으로 분류하는 모델을 활용한 앱**을 개발했습니다. 해당 모델은 학생의 진로 이해 수준과 질문의 특성을 반영해 상담 배정을 보다 합리적으로 지원하는 것을 목표로 합니다. 결과적으로 **교사는 자신이 대응하기 애매한 상담은 미리 걸러내어 상담 업무 부담을 줄이고, 학생은 보다 적합한 교사와 상담할 수 있어 시간과 자원을 절약**할 수 있습니다.

## 프로젝트 구조
```
WhoIsMyCounselor/
├── prototype/
│   ├── app/
│   │   ├── core/              # 핵심 로직
│   │   │   ├── textproc.py    # 학생 정보 및 summary 추출
│   │   │   └── inference.py   # 모델 로드 및 추론 관련 코드
│   │   ├── infra/             # 설정/상수
│   │   │   └── config.py
│   │   ├── ui/                # UI 레이어
│   │   │   ├── shared.py      # 환경 설정 레이어
│   │   │   ├── student_view.py# 학생 패널 레이어
│   │   │   └── teacher_view.py# 교사 패널 레이어
│   │   └── __init__.py
│   ├── main.py                # 내부 엔트리 함수(main) — run_app.py에서 호출
│   ├── run_app.py             # 실제 실행 스크립트
│   ├── merger_model/          # (로컬 전용, .gitignore 대상) 모델 파일 위치
│   ├── app_readmd.md          # 앱 동작 관련 readme
│   └── app_requirements.txt   # 앱 동작 관련 requirements
├── .gitignore
├── LICENSE
└── README.md                  # 전체 프로젝트 설명

```
